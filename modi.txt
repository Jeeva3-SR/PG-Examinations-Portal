PG Examinations Portal - Session Timetable Modifications
=========================================================
Date: 11-Jun-2025
Updated: 11-Jun-2026 (MVC architectural migration from master)

FUNCTIONALITIES ADDED
---------------------

1. Timetable upload (XLSX / CSV)
   - Existing support retained for .xlsx and .csv files.
   - Maximum upload size: 5 MB.

2. Preview-before-commit workflow
   - New endpoint: POST /api/sessions/upload/preview
     Extracts timetable data without writing to the database.
   - New endpoint: POST /api/sessions/upload/commit
     Saves reviewed/edited session rows to the database.
   - Existing endpoint POST /api/sessions/upload remains for direct commit (backward compatible).

3. Course code and name extraction display
   - After extraction, detected course codes and names are shown below the upload area.
   - Entries with missing names are flagged as "Needs manual review".
   - Amber highlighting in the review table indicates rows requiring edits.

4. Editable review table before commit
   - Users can edit date, day, session, course code, course name, specialization, and department.
   - Users can add or remove rows before final commit.
   - Commit is blocked until all required fields are filled.

5. Duplicate session prevention
   - Unique database index on date + session slot + course code + department (active sessions).
   - Duplicate checks on manual add, edit, and timetable commit.
   - API returns 409 Conflict when a duplicate is detected.

6. Department name required
   - Department field added to session model and all create/update flows.
   - Department dropdown in add session form, upload preview table, and edit mode.

7. Editable session list
   - Session list rows can be edited inline (date, day, session, course, specialization, department).
   - Save updates via PATCH /api/sessions/:id.

8. Pre-pone / Postpone with mandatory reason
   - When date or session slot is changed during edit, a modal requires:
     - Pre-pone or Postpone selection
     - Written reason before saving
   - Reschedule metadata stored on the session (originalDate, rescheduleType, rescheduleReason, rescheduledAt).
   - Rescheduled sessions show a badge in the session list.

9. Notifications to assigned people
   - On pre-pone/postpone: email notification sent to:
     - Invigilators assigned for that date/session (Duty)
     - QP setters assigned to the course (AssignedQPSetter)
     - Faculty assigned to the subject (SubjectAssignment)
   - On session delete: same assigned people are notified of cancellation.
   - Uses existing nodemailer utility (console fallback when SMTP not configured).

10. Session delete cleanup and confirmation
   - GET /api/sessions/:id/delete-impact returns student, seating, and duty counts.
   - If session has seating or duties, delete/cancel confirmation warns accordingly.
   - StudentInput records are **preserved** on session cancel/delete (not removed).
   - SeatingArrangement entries and Duty assignments for the slot are still released/freed.
   - Confirmation messages state that student input records will be kept.

11. Duplicate prevention (client + server)
   - Unique key: date + session slot + department + course code (active sessions only).
   - UTC day-range matching fixes timezone-related duplicate bypass.
   - Client-side duplicate check before add/edit with alert:
     "This session entity is already present..."
   - Department column shown before Course Code in session list and upload preview.

12. Session status colors (based on system date)
   - Completed (past date): light green row
   - Pre-poned / Postponed: blue row
   - Scheduled (upcoming): normal white row
   - Cancelled: red row with status badge
   - Status column added to session list with color legend

13. Cancel vs Delete
   - Cancel (POST /api/sessions/:id/cancel): soft-cancel, keeps row in red,
     preserves student inputs, releases seating, frees duties, notifies faculty
   - Delete: permanently removes session from database (student inputs preserved)

14. Frontend dev proxy fix
   - Added frontend/src/setupProxy.js to proxy only /api to backend.
   - Removed blanket package.json proxy that forwarded webpack HMR files
     (e.g. main.*.hot-update.json) to the backend and caused 404 logs.
   - Backend 404 handler now silently ignores hot-update/sockjs dev assets.

15. Session list view filters
   - Filter buttons: All, Completed, Future, Cancelled
   - Completed: past-date sessions (excluding cancelled)
   - Future: today and upcoming sessions (excluding cancelled)
   - Cancelled: soft-cancelled sessions only
   - Shows count: "Showing X of Y session(s)"

16. Session Print / PDF Report
   - "Print / PDF Report" button on Session List page.
   - Filter options before generating: All, Completed, Scheduled, Cancelled
     OR custom date range (from / to).
   - Report header: aulogo (left), ANNA UNIVERSITY / Chennai / Report of Session (center),
     ceglogo (right) from frontend/public/logo.
   - Tabular report with checkbox columns: Students, Invigilators, Seating.
   - Footer: logged-in user email (from useAuthStore) and report generation timestamp.
   - Print and Download PDF actions from preview.
   - GET /api/sessions/report/assignments for assignment status lookup.

17. Department display and update fixes
   - GET /api/sessions returns enriched department + displayStatus
   - PATCH explicitly saves department with parseSessionDate for consistent dates
   - DepartmentSelect preserves custom/existing department values on edit

18. Student input without timetable session (Jun 2026)
   - Student input can be added with manual Exam Date + Session (FN/AN) fields.
   - sessionRef is optional; missing/deleted session no longer blocks create.
   - Only cancelled sessions block new input when a valid sessionRef is linked.
   - MVC: logic in studentInputController.js; thin studentInputRoutes.js.

19. Delete student input entry (Jun 2026)
   - Cross (✕) icon on each row in Submitted Entries table.
   - Calls DELETE /api/student-inputs/:id via existing studentInputController.deleteInput.

20. Session report PDF — portrait A4 (Jun 2026)
   - Print and Download PDF use A4 portrait (210mm × 297mm).
   - Report document sized to A4 width; compact table font for portrait layout.
   - SessionReportModal.jsx updated (no backend change).

21. Sticky pencil edit column — Session List (Jun 2026)
   - First column in session table: pencil icon to edit (replaces text Edit link).
   - Column uses CSS sticky left-0 so it stays visible when scrolling horizontally.
   - Save (✓) and cancel (✕) icons shown in sticky column while editing.
   - SessionView.jsx updated.

22. Test data files (Jun 2026)
   - test/sample_timetable.csv — session timetable upload (5 sessions).
   - test/student_lists/sample_ceg_regular.csv (+ arrear, mit_regular, mit_arrear).
   - test/generate_test_excels.js — generates .xlsx from CSV (run after npm install).
   - test/README.md — how to use each file.

HOW TO TEST
-----------
1. Install backend dependencies:
   cd backend && npm install

2. Run unit tests:
   cd backend && npm test

3. Manual UI test:
   - Start backend and frontend servers.
   - Log in as coordinator.
   - Open Coordinator > Session Timetable page.
   - Upload a timetable CSV/XLSX, set department for each row, then commit.
   - Try adding a duplicate session — should be blocked.
   - Edit a session and change the date — pre-pone/postpone modal should appear.
   - Provide a reason and save — check backend console for notification emails.
   - Cancel or delete a session — student inputs preserved; seating/duties freed.
   - Open Print / PDF Report — portrait A4 PDF; footer shows login email + timestamp.
   - Student Input: add entry with manual date/session; delete with ✕ icon.
   - Upload student lists using test/student_lists/*.xlsx (generate via test script).
   - See test/README.md for full test file guide.
